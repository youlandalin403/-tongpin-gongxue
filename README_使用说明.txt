同频共学 README_使用说明.txt
版本：最终提交演示版
项目名称：同频共学——面向大学生的兴趣共学与学习陪伴网页平台


一、作品简介
同频共学是一个面向大学生校内使用的兴趣共学网页平台。平台围绕“兴趣发现—小组共学—资源收录—每日打卡—交流反馈—成长记录”的完整学习闭环展开，帮助学生找到同频学习伙伴，沉淀优质学习资源，并通过打卡和小组交流提升学习持续性。

本版本为本地可运行演示版，适用于课程实践、项目展示和功能演示。网页无需部署服务器，解压后即可在本地浏览器中运行。


二、运行环境
1. 操作系统：Windows 10 / Windows 11
2. 浏览器：推荐使用 Microsoft Edge、Google Chrome 或其他现代浏览器
3. 是否需要联网：不需要
4. 是否需要安装数据库：不需要
5. 是否需要安装 Python / Node.js / Java：不需要

说明：
本版本使用浏览器本地存储模拟用户、登录状态、小组加入、资源收藏、资源收录、打卡和交流数据，因此可以直接本地运行。真实上线时可将本地存储逻辑替换为后端接口与数据库。

四、启动方式
方式一：推荐启动方式
1. 解压“同频共学_最终提交版.zip”。
2. 双击根目录下的“双击启动.bat”。
3. 系统会自动打开浏览器并进入同频共学平台。

方式二：手动启动方式
1. 打开“网页演示版”文件夹。
2. 双击“启动同频共学.html”。
3. 如果仍未打开，可直接双击“index.html”。

方式三：浏览器手动打开
1. 打开浏览器。
2. 将“网页演示版/index.html”拖入浏览器窗口。
3. 页面加载完成后即可使用。

六、主要功能说明
1. 首页
展示平台定位、今日共学状态、推荐入口和核心功能导航。用户可以从首页进入兴趣地图、小组、资源、打卡、交流和成长档案等模块。

2. 兴趣地图
兴趣地图是平台的核心入口之一。页面中展示多个彩色兴趣方块，例如考试学习、专业技能、科研提升、竞赛实践、语言提升、审美创造等。点击兴趣方块后，会展开更细的方向、推荐小组和行动建议。

3. 共学小组
用户可以浏览不同学习小组，查看小组方向、人数、打卡周期和小组简介。用户可以加入小组，也可以创建新的共学小组。加入小组后，相关信息会保存到当前登录账号下。

4. 资源书院
资源书院用于收录和管理学习资源。平台收录的是“资源说明 + 可访问链接”，不是直接上传大文件。用户可以搜索资源、按方向筛选资源、查看资源详情、收藏资源，也可以收录新的学习资源。

资源类型示例：
- 视频课程
- 文档资料
- 工具网站
- 经验文章
- 练习题库
- 学习模板
- 复盘清单

5. 今日打卡
用户可以选择所属小组，填写学习主题、学习时长、学习内容、完成状态和心情反馈。提交后，打卡记录会显示在打卡墙中，并绑定当前用户。

6. 交流庭院
用户可以在交流区发布学习问题、经验复盘或小组讨论内容，也可以点赞和回复他人的帖子。该模块用于模拟小组内部的公开交流场景。

7. 成长档案
成长档案展示当前用户的学习统计，包括累计打卡、连续打卡、学习时长、加入小组、收藏资源等信息，用于呈现用户的学习轨迹和成长记录。

8. 智能学伴
智能学伴可以根据用户选择的学习方向、时间安排和当前状态，生成今日学习建议。该功能用于展示平台后续可扩展的智能学习计划推荐能力。


七、推荐演示流程
正式展示或录制视频时，建议按照以下流程操作：

1. 打开网页，进入登录页。
2. 使用 demo / 123456 登录，或注册一个新账号。
3. 进入首页，简单介绍平台目标。
4. 进入兴趣地图，点击一个兴趣方块，展示细分方向和推荐小组。
5. 进入共学小组，搜索或筛选小组，并加入一个小组。
6. 进入资源书院，筛选资源、查看资源详情、收藏资源。
7. 点击“收录资源”，新增一条资源记录。
8. 进入今日打卡，提交一次学习打卡。
9. 进入交流庭院，发布或回复一条讨论内容。
10. 进入成长档案，查看学习统计。
11. 进入智能学伴，生成一份今日学习建议。
12. 退出登录，结束演示。


八、数据保存说明
本版本为前端本地演示版，数据保存在浏览器 localStorage 中。不同浏览器之间数据不互通，同一浏览器清除缓存后，注册用户、打卡记录、收藏资源、收录资源等数据可能会被清空。

本地演示版保存的数据包括：
1. 注册用户信息
2. 当前登录状态
3. 用户加入的小组
4. 用户收藏的资源
5. 用户收录的新资源
6. 用户提交的打卡记录
7. 用户发布、点赞和回复的交流内容

真实上线时，以上数据应由后端服务写入数据库。


九、部署说明
本版本无需服务器部署，直接本地打开即可运行。

如果后续需要部署到服务器，可采用以下方式：
1. 将“网页演示版”中的所有文件上传到服务器静态目录。
2. 使用 Nginx、Apache 或校园服务器提供静态网页访问。
3. 将本地存储逻辑改为后端接口调用。
4. 后端可使用 Spring Boot、Flask、Django 或其他框架。
5. 数据库可使用 MySQL、SQLite 或 PostgreSQL。
6. 用户、兴趣标签、小组、资源、打卡、帖子和评论等数据应存入数据库。

建议第一阶段保持本地可运行，第二阶段再进行前后端联调和服务器部署。

十一、注意事项
1. 请先解压压缩包后再运行，不建议直接在压缩包预览界面中打开网页。

十三、项目定位说明
本项目不是普通聊天社区，也不是单纯资料库，而是一个围绕大学生学习目标组织起来的兴趣共学平台。它通过兴趣地图帮助用户发现方向，通过共学小组组织同频用户，通过资源书院沉淀资料，通过每日打卡促进持续行动，通过交流庭院形成反馈，通过成长档案展示学习轨迹。
